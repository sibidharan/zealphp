## What's new in 0.4.0 (the headlines)

### 1. `$req` / `$res` — short injection aliases
Handlers can name params `$req`/`$res` instead of `$request`/`$response` — works in routes, api, fallback/error, and template closures.
```php
$app->route('/hello/{name}', fn ($name, $req) =>
    ['hello' => $name, 'htmx' => $req->isHtmx()]);
```
An explicit `{req}` URL segment still wins; `$request`/`$response` keep working.

### 2. `App::renderHtmx()` + the htmx story (the 0.4.0 feature)
The framework already had **full htmx support** — 8/8 request accessors (`$req->isHtmx()`, `htmxTarget()`, …) and 11/11 response headers via `$res->htmx()` (`retarget`, `reswap`, `trigger`, …). 0.4.0 adds the **missing ergonomic helper**:
```php
// Before: hand-rolled every time
if ($req->isHtmx()) return App::render('contacts', ['fragment' => 'list']);
return App::render('contacts');

// After: one line — fragment on an htmx request (auto from HX-Target), full page otherwise
return App::renderHtmx('contacts');
```
Plus `$res->htmx()->triggerJSON('saved', ['id' => 7])` and a chain-back `->response()`.
**New consolidated HTMX guide** → `/docs/guide/htmx` (in the docs index + sidebar + nav).

### 3. All CGI env vars (two-tier worker model)
The CGI **back-of-house** subprocess pool is now fully env-tunable (was code-only):
`ZEALPHP_CGI_WORKERS` (pool size, FPM `pm.max_children`), `ZEALPHP_CGI_MAX_REQUESTS`,
`ZEALPHP_CGI_TIMEOUT`, `ZEALPHP_FCGI_ADDRESS`, `ZEALPHP_CGI_FORK_MAX_CONCURRENT`.
> **Teaching point — two pools:** front-of-house = OpenSwoole workers (`ZEALPHP_WORKERS`); back-of-house = CGI subprocess pool (`ZEALPHP_CGI_WORKERS`, *per worker* → total = `workers × cgi_workers`). Only exists in `legacy-cgi` / processIsolation mode.

### 4. Foreground start banner
`php app.php` (no `-d`) used to start silently; now prints
`ZealPHP running at http://localhost:8080  (N routes, W workers)  —  press Ctrl+C to stop`.

### 5. Redesigned scaffold (`composer create-project sibidharan/zealphp-project`)
Terminal-Luxury theme + a live **htmx playground** — 5 panels, each a real handler.

## Concepts you were exploring (for the lecture)

- **Mixed mode = PHP-FPM's exact execution model.** `App::mode('mixed')` = `superglobals(true)` + scheduler off + in-process. Sequential per warm worker, native `$_GET/$_POST/$_SESSION`, minus the FastCGI hop + separate web server. Workers = concurrency (size like `pm.max_children`).
- **coroutine-legacy** isolates the 7 superglobals **including `$_SESSION`** per coroutine (needs ext-zealphp) — so `$_SESSION` *and* `$g->session` both work, per-request.
- **Header parity:** OpenSwoole lowercases header keys; ZealPHP rebuilds them to the CGI `HTTP_UPPERCASE` convention in `$_SERVER`/`$g->server`, so `$_SERVER['HTTP_USER_AGENT']` works (Apache parity).
